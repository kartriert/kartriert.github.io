Website

Eine wichtige Frage, die man beim Erstellen einer Website klären sollte, ist, ob man eine dynamische oder statische Website erstellen möchte. Dynamische Websites werden benutzt, wenn viel mit Nutzer:innen interagiert wird, wie bei einem Online-Shop zum Beispiel. Da der Fokus unserer Website darauf liegt, Informationen zu präsentieren, und sie keine Daten von Nutzenden sammelt, haben wir uns für eine statische Website entschieden. Dadurch, dass wir GitHub benutzen, um das Projekt und den gesamten Workflow zu verwalten, war es für uns am sinnvollsten, die Website mit GitHub Pages zu erstellen und so direkt in unser GitHub-Projekt einzubinden.


Für das Design und die Optik der Website haben wir uns für ein vorgefertigtes Template entschieden, bei dem wir die verschiedenen Elemente (wie zum Beispiel die die Akkordeon-Funktion auf der Seite "Über Uns", die Anordnung von Elementen und deren Farben durch Bearbeitung der jeweiligen CSS-Dateien angepasst haben. Die Website wird auch über ein GitHub-Repository verwaltet und ermöglicht so das Einbinden von Daten und, in unserem Fall, das Einbauen einer interaktiven Karte. Sonstige Inhalte, wie etwa dieser Text, wurden direkt in die HTML-Dateien geschrieben. Das verwendete Hintergrundbild auf der Startseite ist von Jakob Braun (kostenlos zu verwenden im Rahmen der Unsplash Lizenz).

Für das Logo kam die Idee auf, eine Kutsche darstellen zu wollen, das Haupttransportmittel der damaligen Zeit auch für größere Distanzen. Es wurden also Vorschläge gesammelt und mehrere Entwürfe erstellt. Schließlich haben wir uns dafür entschieden, als kleines Logo eine Kutsche auf einem Geopointer und zusätzlich eine größere Version einer Kutsche mit vorangespanntem Pferd für die Startseite zu erstellen.
